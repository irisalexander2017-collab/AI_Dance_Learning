# 帧琢 FrameTune 项目说明（Codex 开发版）

## 1. 项目名称

**帧琢 FrameTune**（代码仓库：`AI_Dance_Learning`）

当前公开测试版的用户功能全部在浏览器中运行。用户选择的舞蹈视频、学习者摄像头画面和生成的练习录像均保留在本机，不会上传。摄像头只在用户主动开启时显示；MVP-08 支持用户主动录制教师与学习者的左右分屏，不录制麦克风或环境声音，可在本地回看、下载 WebM 或删除。

## 2. 项目目标

本项目不是复杂的动作识别或实时纠错系统。

第一版的核心目标是：

> 把一段 15–60 秒的舞蹈视频，自动变成可以按八拍、慢速、镜像、循环、局部放大练习的互动播放器。

主要解决的问题：
- 用户在 YouTube、Instagram、TikTok、微信视频号或课堂录像中看到一支舞，但没有老师现场教学；
- 非专业舞者需要花很长时间反复拖动视频、慢放、倒回和数拍；
- 快速手部动作和复杂脚步很难看清；
- 学完一个八拍后，需要快速把前面的动作连起来练习；
- 学舞时不适合频繁操作电脑，因此常用功能必须尽量一键完成。

## 3. 第一版用户输入

用户只需要在浏览器中选择本地舞蹈视频；文件不会上传到服务器。

视频来源可能包括：
- YouTube 屏幕录制；
- Instagram、TikTok 或微信视频号保存/录制的视频；
- 舞蹈课堂现场录像；
- 老师示范视频；
- 手机拍摄的视频。

当前界面接受 MP4、MOV、WebM；能否播放仍取决于浏览器是否支持文件内部的编码。

当前公开测试版没有后台转码，也不依赖 FFmpeg。遇到浏览器不支持的编码时，需要先把视频转换为浏览器可播放的格式。

## 4. 核心产品逻辑

### 4.1 自动分析音乐

系统从视频中自动提取音频并分析：
- BPM；
- 每一个拍点的位置；
- 音乐中的 1–8 八拍结构；
- 尝试识别音乐的第一拍；
- 节拍识别置信度。

系统的自动分析范围仅限音乐拍点、BPM 和音乐八拍结构。系统不得自动判断正式编舞从哪里开始，因为老师可能在正式编舞前加入 freestyle、准备动作或临时动作。

第一版优先支持节拍清晰、已经配好音乐的社交平台视频。课堂现场视频可以自动尝试分析，但必须保留人工校准方案。

### 4.2 第一拍校准

系统自动提出第一拍以后，用户可以：
- 确认正确；
- 向前或向后调整一拍；
- 向前或向后调整半拍；
- 自己点击“这里是第一拍”。

如果自动识别失败，用户可以跟着音乐点击连续 4 拍或 8 拍，系统根据点击结果建立拍点时间线。

### 4.3 正式编舞开始点

用户在视频播放过程中，只需点击一次“正式编舞从这里开始”。

系统把用户点击时间自动对齐到最近的音乐整数拍或半拍，并根据音乐时间线判断：
- 正式编舞从第几个八拍开始；
- 正式编舞从该八拍的第几拍进入；
- 或是否从某一拍的 “and” 进入。

系统从用户确认的位置开始划分学习段，并自动生成正式动作开始前四拍的预备计数。正式学习起点之前的视频继续保留，可通过“播放完整原视频”查看，但默认不进入学习八拍。

音乐时间线始终保留真实拍点；学习段从用户选择的正式编舞起点开始重新编号。重新编号只影响学习段的显示与练习顺序，不改变原始音乐拍点。

系统不需要识别动作本身，也不得自行推断正式编舞起点；系统只负责把用户点击的正式编舞起点与音乐拍点对应起来。

此功能属于后续阶段，当前仅记录需求，不在 Phase 0 或 Phase 1 中实现。

### 4.4 自动预备拍

默认规则：动作开始前提供四拍预备。

| 动作进入拍 | 默认预备拍 |
|---|---|
| 第1拍 | 前一八拍的 5、6、7、8 |
| 第2拍 | 前一八拍的 6、7、8，加当前 1 |
| 第3拍 | 前一八拍的 7、8，加当前 1、2 |
| 第4拍 | 前一八拍的 8，加当前 1、2、3 |
| 第5拍 | 当前 1、2、3、4 |
| 第6拍 | 当前 2、3、4、5 |
| 第7拍 | 当前 3、4、5、6 |
| 第8拍 | 当前 4、5、6、7 |

系统允许切换：
- 固定提前四拍；
- 从当前八拍第1拍开始数；
- 只数整数拍；
- 数 1-and-2-and；
- 只播放预备拍；
- 关闭数拍。

## 5. 八拍学习模式

系统不把视频真正切成多个文件，而是保留原始完整视频，只保存每个八拍的开始和结束时间。

用户正在学习第 4 个八拍时，应可一键选择：
- 只练第 4 个八拍；
- 第 3 + 第 4 个八拍；
- 第 2 + 第 3 + 第 4 个八拍；
- 从头练到第 4 个八拍；
- 整支舞；
- 上一个八拍；
- 下一个八拍。

切换练习范围时，不能重新加载视频或重置播放位置。

## 6. 自动循环

用户选择练习范围以后，默认自动循环。

循环选项：
- 连续循环；
- 每次停 2 秒；
- 每次停 4 秒。

建议默认：提前播放 1 拍过渡、停顿 2 秒、播放四拍预备、自动开始当前练习段。

## 7. 播放控制

必须支持一键操作：
- 播放 / 暂停；
- 0.5×、0.65×、0.8×、1×；
- 镜像；
- 回看 2 秒；
- 前进 2 秒；
- 上一帧；
- 下一帧；
- 全屏。

镜像切换时，视频不能重新加载，播放位置不能重置，局部放大画面必须同步镜像。

## 8. 局部放大功能

用户应当能够一键选择：
- 全身；
- 脚部；
- 手部；
- 上半身；
- 点哪里，放大哪里。

第一版实现：
- 预设脚部、手部、上半身区域；
- 用户点击视频任意位置；
- 以点击位置为中心生成局部放大画中画；
- 放大画面与主视频同步播放、暂停、慢放、循环和镜像；
- 一键恢复全身。

第二版再做：自动追踪脚部和手部。第二版可以轻量使用 MediaPipe，但仅用于部位跟踪，不用于动作评分。

## 9. 键盘快捷键

| 按键 | 功能 |
|---|---|
| Space | 播放 / 暂停 |
| Left | 回退 2 秒 |
| Right | 前进 2 秒 |
| Up | 上一个八拍 |
| Down | 下一个八拍 |
| M | 镜像 |
| L | 开关循环 |
| F | 脚部放大 |
| H | 手部放大 |
| R | 从头练到当前 |

## 10. 保存练习进度

系统需要记住同一个视频的：
- 第一拍校准结果；
- 动作开始点；
- 当前学到第几个八拍；
- 当前播放速度；
- 是否镜像；
- 当前数拍模式；
- 当前循环模式；
- 当前局部放大模式。

第一版本地运行时，可先使用本地 JSON、SQLite 或浏览器 Local Storage，不需要用户登录。

## 11. 练习页面设计原则

练舞过程中不能频繁操作。所有高频按钮必须：
- 足够大；
- 固定位置；
- 一键完成；
- 不需要多层菜单；
- 不需要拖动复杂时间线；
- 不需要理解技术参数。

页面结构：

### 顶部
- 主视频；
- 局部放大画中画。

### 中部
显示当前状态，例如：

> 第 3 个八拍 · 0.65× · 循环中 · 脚部放大

### 底部
大按钮：
- 上一个；
- 播放 / 暂停；
- 下一个；
- 当前八拍；
- 前一个 + 当前；
- 从头到这里；
- 整支舞；
- 全身；
- 脚部；
- 手部；
- 点选放大；
- 0.5×；
- 0.65×；
- 0.8×；
- 原速；
- 镜像；
- 数拍；
- 回看 2 秒。

## 12. 第一版不做的功能

禁止加入：
- 动作评分；
- 用户实时摄像头纠错；
- 动作名称识别；
- 舞种识别；
- 自动解释动作；
- 多人舞蹈识别；
- YOLO；
- 复杂骨架可视化；
- 多智能体框架；
- CrewAI；
- LangChain Agent；
- LLM 读取完整 Pose JSON；
- GPT / Claude 视频理解；
- 用户登录；
- 用户账号、云端保存或上传私人练习资料；
- 社交功能；
- 课程商城；
- 订阅和付费系统。

第一版不需要 LLM API。

## 15. 开发阶段

### Phase 0：项目初始化
- 建立文件结构；
- 初始化 Git；
- 创建 README、开发日志和测试清单；
- 创建最简单首页；
- 确认前后端可以启动。

### Phase 1：基础播放器
- 上传视频；
- 自动格式转换；
- 播放 / 暂停；
- 四档速度；
- 镜像；
- 回退 2 秒；
- 全屏；
- 键盘快捷键。

### Phase 2：手动八拍练习
- 用户手动标记第一拍；
- 用户输入或选择 BPM；
- 系统生成拍点和八拍；
- 当前八拍循环；
- 前一个 + 当前；
- 从头到当前；
- 整支舞；
- 自动预备拍。

### Phase 3：自动音乐节拍分析
- 提取音频；
- 检测 BPM 和拍点；
- 尝试识别第一拍；
- 自动生成八拍；
- 支持校准；
- 失败时回到手动模式。

### Phase 4：动作起点与预备拍
- 用户点击一次“正式编舞从这里开始”；
- 将点击时间对齐到最近的音乐整数拍或半拍；
- 判断正式动作从第几拍或哪个 “and” 进入；
- 从对齐位置建立学习段并自动生成四拍预备；
- 保留学习起点之前的视频，并支持“播放完整原视频”；
- 音乐时间线保留真实拍点，学习段从正式起点重新编号；
- 不自动判断正式编舞起点。

### Phase 5：局部放大
- 脚部、手部、上半身；
- 点选放大；
- 画中画；
- 同步慢放、镜像和循环；
- 一键恢复。

### Phase 6：练舞体验优化
- 循环停顿；
- 逐帧查看；
- 自动保存进度；
- 大按钮；
- 手机和平板适配；
- 防误触；
- 隐藏技术参数。

### Phase 7：本地可用版本
- 一键启动；
- 错误提示；
- 历史舞蹈列表；
- 删除项目；
- 重新分析；
- 导出和备份；
- Windows 本地运行说明。

## 16. Codex 开发规则

Codex 每次只完成一个阶段或一个明确模块，禁止一次性实现整个项目。

每次任务必须遵守：
1. 不修改与当前任务无关的文件；
2. 不引入未经批准的新框架；
3. 不改变项目范围；
4. 不加入第一版禁止功能；
5. 完成后必须运行测试；
6. 列出修改过的文件；
7. 提供启动命令；
8. 记录已知问题；
9. 更新 DEVELOPMENT_LOG.md；
10. 完成可用阶段后创建 Git commit。

建议 commit：

```text
MVP-00: project setup
MVP-01: video upload and playback
MVP-02: manual eight-count practice
MVP-03: automatic beat detection
MVP-04: action start and count-in
MVP-05: focus zoom and picture-in-picture
MVP-06: practice UX improvements
MVP-07: local release
MVP-08: local practice recording with teacher audio
```

## 17. Codex 第一条任务

> 初始化 AI_Dance_Learning 项目，建立 React + TypeScript + Vite 前端和 Python FastAPI 后端。建立推荐文件结构、README、开发日志、测试清单和 Git 仓库。创建一个可以打开的基础首页，并提供 Windows 下的启动脚本。暂时不要实现视频上传、节拍分析、MediaPipe、LLM、多智能体或其他业务功能。完成后运行前后端测试，列出所有创建和修改的文件，并创建 Git commit：`MVP-00: project setup`。

## 18. 第一版成功标准

第一版成功，不是因为系统“很智能”。

第一版成功的标准是：

> 用户上传一段短舞蹈视频后，可以快速按音乐八拍学习，随时慢放、镜像、循环、串联前面动作，并一键放大脚部或手部，不需要频繁拖动视频或处理音频文件。
